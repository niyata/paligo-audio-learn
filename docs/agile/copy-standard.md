# Paligo UI Copy Standard (Apple HIG)

อ้างอิง issue #71 — ใช้กับข้อความที่ผู้ใช้เห็นในแอป (ไม่รวม landing/marketing)

## หลักการ

- **สั้น ชัด** — ไม่อธิบาย flow ยาวในปุ่มหรือหัวข้อ
- **Action-oriented** — ปุ่มใช้คำกริยา: บันทึก, ส่งตรวจ, เปิดสมุด, ยกเลิก
- **Consistent** — คำเดียวกันทุกหน้า (ดู glossary)
- **Accessible** — ปุ่มไอคอนล้วนต้องมี `aria-label` ภาษาไทย; state ใช้ `aria-pressed` / `aria-selected`

## Glossary (คำมาตรฐาน)

| คำ | ใช้เมื่อ | หลีกเลี่ยง |
|----|---------|-----------|
| สมุดข้อสอบ | รายการ, เมนู, empty state, fallback ชื่อ | สมุดคำตอบ (ยกเว้นหัวปกที่พิมพ์ `สมุดคำตอบ ป.ธ.` บนกระดาษ) |
| เปิดสมุด | ปุ่มเปิดเล่ม | เปิดเล่ม |
| สร้างสมุดใหม่ | สร้างเล่มใหม่ | |
| ส่งตรวจ | ส่งให้ครูตรวจ | ส่ง inbox, ส่งโทเค็น |
| ยกเลิกส่งตรวจ | ถอนการส่ง | ยกเลิกการส่ง |
| ผลตรวจ | คะแนน/ไฟล์จากครู | |
| กล่องข้อความ | หัวข้อหน้า, ปุ่มนำทาง | Inbox (ยกเว้นแท็บบัญชีเทคนิค) |
| บัญชี | หน้า login/สมัคร | |
| จับคู่ | ครู–นักเรียน | |
| โอนไฟล์ขั้นสูง | import/export สำรอง | |

## State patterns

| สถานะ | ข้อความ |
|-------|---------|
| Loading | `กำลังโหลด…` / `กำลังบันทึก…` / `กำลังส่ง…` |
| Success | `บันทึกแล้ว` / `ส่งตรวจแล้ว` |
| Offline | `ออฟไลน์ — แสดงข้อมูลในเครื่อง` |
| Empty (สมุด) | `ยังไม่มีสมุดข้อสอบ` + ปุ่ม `สร้างสมุดใหม่` |

## ปุ่ม

- Primary = action หลักของ modal/หน้า
- Destructive = คำตรง: ทิ้งสมุด, ลบรูป, ออกจากระบบ
- ยืนยัน destructive = บอกผลกระทบ 1–2 บรรทัด + ว่าข้อมูลยังอยู่ในเครื่อง (soft delete)

## ไม่แก้ในงาน copy

- Schema keys, API fields, storage keys, enum values
- ชื่อไฟล์ `.paligo-review.json` ในคำแนะนำเทคนิค
- หัวปกสมุด `สมุดคำตอบ ป.ธ. …` (รูปแบบบนหน้ากระดาษ)
