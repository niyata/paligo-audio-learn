# PRD และ Roadmap: แอปเรียนภาษาบาลีในประเทศไทย

## ภาพรวมผลิตภัณฑ์

แอปพลิเคชันนี้มุ่งช่วยผู้เรียนภาษาบาลีในประเทศไทย โดยยึดบริบทของการศึกษาพระปริยัติธรรมแผนกบาลี ซึ่งมีหลักสูตร ตำรา และรูปแบบการสอบที่เฉพาะเจาะจงมาก ทั้งสำหรับพระภิกษุสามเณรในระบบเปรียญธรรม และฆราวาสในระบบบาลีศึกษา

เป้าหมายหลักคือช่วยลดภาระการท่องจำ เพิ่มความเข้าใจโครงสร้างภาษา และทำให้การฝึกแปลบาลีเป็นไทยหรือไทยเป็นบาลีเข้าถึงง่ายขึ้นบน iOS

## กลุ่มผู้ใช้เป้าหมาย

- พระภิกษุสามเณรที่เรียนเปรียญธรรม ป.ธ. ๑-๙
- ฆราวาสที่เรียนบาลีศึกษา บ.ศ. ๑-๙
- ครูสอนบาลี สำนักเรียน และผู้ช่วยติวข้อสอบ
- ผู้สนใจศึกษาพระไตรปิฎก อรรถกถา และตำราบาลีด้วยตนเอง

## บริบทหลักสูตรและตำราเรียน

### ชั้นประโยค ๑-๒ และประโยค ๓

ช่วงนี้เป็นรากฐานสำคัญที่สุดของผู้เรียนบาลีไทย

- ตำราไวยากรณ์: บาลีไวยากรณ์ พระนิพนธ์ของสมเด็จพระมหาสมณเจ้า กรมพระยาวชิรญาณวโรรส
- หมวดเนื้อหาหลัก: นาม/อัพยยศัพท์, อาขยาต/กิตก์, สมาส/ตัทธิต, สมัญญาภิธาน/สนธิ
- ตำราแปลมคธเป็นไทย: ธัมมปทัฏฐกถา ภาค ๑-๘
- วิชากลับสำหรับ ป.ธ. ๓: ฝึกแต่งภาษาไทยกลับเป็นภาษาบาลี

### ชั้นประโยค ๔-๖

ผู้เรียนเริ่มเข้าสู่ตำราที่ซับซ้อนและต้องใช้ทักษะวิเคราะห์ประโยคมากขึ้น

- ตำราแปล: มังคลัตถทีปนี
- ตำราแปล: สมันตปาสาทิกา

### ชั้นประโยค ๗-๙

เป็นระดับสูงสุด เน้นความแม่นยำในการแปล แต่ง และวิเคราะห์เนื้อหาชั้นสูง

- ตำราแปลและแต่ง: วิสุทธิมรรค
- ตำราอภิธรรม: อภิธัมมัตถสังคหะ

## Pain Points ของผู้เรียน

- ต้องท่องจำศัพท์ ธาตุ วิภัตติ และกฎไวยากรณ์จำนวนมาก
- หนังสือตารางแจกศัพท์และตำราอ้างอิงใช้งานบนมือถือไม่สะดวก
- การแยกโครงสร้างประโยคบาลีทำได้ยาก โดยเฉพาะผู้เริ่มต้น
- การค้นคำบาลีมักต้องรู้รูปศัพท์ตั้งต้นก่อน จึงค้นได้แม่น
- ข้อสอบเก่าและเฉลยกระจายอยู่หลายแหล่ง
- ผู้เรียนขาดระบบติดตามความก้าวหน้าและการทบทวนแบบเป็นรอบ

## ฟีเจอร์หลัก

### 1. Interactive Declension & Conjugation Tables

ตารางแจกวิภัตติและการผันกริยาแบบ Interactive สำหรับนามศัพท์และอาขยาต

ความสามารถหลัก:

- ค้นหรือเลือกศัพท์ตัวอย่าง เช่น ปุริส, มุนิ
- แสดงรูปแจกตามวิภัตติ ลิงค์ และวจนะ
- แสดงการผันกริยาอาขยาตตามกาล บุรุษ และวจนะ
- รองรับการอธิบายที่มาของรูปศัพท์แบบสั้น

ประโยชน์:

- ลดการพึ่งหนังสือตารางหนา ๆ
- ช่วยผู้เรียนเห็น pattern การเปลี่ยนรูปได้เร็วขึ้น

### 2. Smart Pali-Thai Dictionary

พจนานุกรมบาลี-ไทยแบบ Offline พร้อมระบบค้นหาที่เหมาะกับอักษรบาลีในบริบทภาษาไทย

ความสามารถหลัก:

- ค้นคำบาลีและคำแปลไทยแบบรวดเร็ว
- รองรับอักขระบาลี เช่น นิคหิต พินทุ ฐ ญ และรูปสะกดที่พบบ่อย
- วิเคราะห์รากศัพท์เบื้องต้น เช่น ธาตุ ปัจจัย วิภัตติ
- ตัวอย่าง: ค้นคำว่า "คจฺฉติ" แล้วแสดงว่าเกี่ยวกับ "คมฺ" ธาตุ พร้อมรูปประกอบ

ประโยชน์:

- ผู้เรียนไม่จำเป็นต้องรู้รูปตั้งต้นของคำทั้งหมดก่อนค้น
- ใช้งานได้แม้ไม่มีอินเทอร์เน็ต

### 3. Flashcards & Spaced Repetition

ระบบทบทวนความจำแบบเป็นรอบ สำหรับศัพท์ ธาตุ กฎไวยากรณ์ และอัพยยศัพท์

ความสามารถหลัก:

- สร้างชุดการ์ดตามหมวด เช่น อัพยยศัพท์ ธาตุ สนธิ สมาส
- ใช้หลักการ spaced repetition คล้าย Anki
- บันทึกสถานะความจำและความคืบหน้าของผู้เรียน
- รองรับการ์ดแบบพลิกหน้า-หลังด้วย SwiftUI Animation

ประโยชน์:

- ลดการท่องจำแบบกระจัดกระจาย
- ช่วยให้ผู้เรียนทบทวนก่อนลืม

### 4. Interactive Translation Assistant

หน้าจออ่านและฝึกแปลข้อความบาลี โดยให้ผู้เรียนแตะคำเพื่อดูความหมายและหน้าที่ของคำในประโยค

ความสามารถหลัก:

- แสดงเนื้อหาจากธัมมปทัฏฐกถาเป็นย่อหน้า
- แตะคำบาลีเพื่อดูคำแปล หน้าที่ทางไวยากรณ์ และคำอธิบาย
- แสดงบทบาทของคำ เช่น ประธาน กริยา ส่วนขยาย หรือกรรม
- รองรับการซ่อน/แสดงคำใบ้เพื่อฝึกแปลด้วยตนเอง

ประโยชน์:

- ช่วยให้ผู้เริ่มต้นจับโครงสร้างประโยคได้ง่ายขึ้น
- ใช้เป็นเครื่องมือฝึกแปลในห้องเรียนหรืออ่านเอง

### 5. Mock Exams & Timer

คลังข้อสอบเก่าสนามหลวงพร้อมระบบจับเวลาและเฉลย

ความสามารถหลัก:

- รวมข้อสอบเก่า ป.ธ. ๑-๙ ย้อนหลังตามที่จัดหาได้
- ฝึกทำข้อสอบพร้อมตัวจับเวลา
- บันทึกผลการทำข้อสอบย้อนหลัง
- แสดงเฉลยหรือแนวคำตอบหลังทำเสร็จ

ประโยชน์:

- ช่วยเตรียมสอบอย่างเป็นระบบ
- ทำให้ผู้เรียนคุ้นกับรูปแบบข้อสอบจริง

## Roadmap การพัฒนา

### Phase 1: Foundation & Dictionary

ระยะเวลาโดยประมาณ: เดือนที่ 1

งานหลัก:

- สร้างโครงสร้างแอปด้วย SwiftUI
- ออกแบบฐานข้อมูลด้วย SwiftData หรือ CoreData
- เตรียม data model สำหรับคำศัพท์บาลี คำแปล หมวดหมู่ และ metadata
- สร้างพจนานุกรมบาลี-ไทยแบบ Offline
- นำเนื้อหา "บาลีไวยากรณ์" ทั้ง 4 หมวดหลักเข้ารูปแบบบทความหรือ e-book ที่อ่านง่ายบนมือถือ

ผลลัพธ์ที่คาดหวัง:

- ผู้ใช้ค้นศัพท์พื้นฐานได้
- ผู้ใช้อ่านเนื้อหาไวยากรณ์หลักบนมือถือได้
- แอปมีโครงสร้างข้อมูลพร้อมต่อยอดฟีเจอร์อื่น

### Phase 2: Memorization Tool

ระยะเวลาโดยประมาณ: เดือนที่ 2

งานหลัก:

- พัฒนาระบบตารางแจกวิภัตติสำหรับนามและกริยา
- พัฒนาระบบ Flashcards
- เพิ่ม progress tracking สำหรับสถานะการเรียน
- ใช้ SwiftUI Animation สำหรับการ์ดพลิกและ interaction สำคัญ
- เก็บสถานะด้วย SwiftData หรือ UserDefaults ตามความซับซ้อนของข้อมูล

ผลลัพธ์ที่คาดหวัง:

- ผู้ใช้ฝึกท่องศัพท์และกฎไวยากรณ์ได้เป็นรอบ
- ผู้ใช้เปิดดูตารางแจกศัพท์ได้รวดเร็ว
- แอปเริ่มเป็นเครื่องมือเรียนประจำวันได้

### Phase 3: Translation Assistant & Content

ระยะเวลาโดยประมาณ: เดือนที่ 3-4

งานหลัก:

- นำเข้าเนื้อหาธัมมปทัฏฐกถา ภาค 1-4 สำหรับประโยค 1-2
- สร้าง UI สำหรับอ่านธรรมบทแบบ paragraph
- ทำคำศัพท์ในเนื้อหาให้แตะเพื่อดูความหมายได้
- ออกแบบโครงสร้าง JSON หรือฐานข้อมูลที่ผูกคำบาลีกับคำแปลไทยทีละคำ
- พัฒนา parser หรือเครื่องมือช่วยจัดโครงสร้างข้อความบาลี

ผลลัพธ์ที่คาดหวัง:

- ผู้ใช้ฝึกอ่านและวิเคราะห์ประโยคจากเนื้อหาจริงได้
- ผู้เรียนเริ่มเห็นความสัมพันธ์ระหว่างศัพท์ ไวยากรณ์ และประโยคแปล

### Phase 4: Testing & Exam Prep

ระยะเวลาโดยประมาณ: เดือนที่ 5

งานหลัก:

- เพิ่มคลังข้อสอบเก่าสนามหลวง
- เพิ่มระบบจับเวลาทำข้อสอบ
- เพิ่มเฉลยหรือแนวคำตอบ
- ทดสอบประสิทธิภาพของฐานข้อมูล Offline
- เตรียม App Store screenshots
- เขียน Privacy Policy
- ปล่อยทดสอบผ่าน TestFlight ก่อนขึ้น Production

ผลลัพธ์ที่คาดหวัง:

- แอปพร้อมใช้สำหรับเตรียมสอบ
- มี flow การใช้งานครบตั้งแต่เรียน ท่อง ฝึกแปล และซ้อมสอบ
- พร้อมส่งขึ้น App Store

## แนวทางเทคนิคเบื้องต้น

### Platform

- iOS
- Swift
- SwiftUI

### Data Storage

- SwiftData สำหรับข้อมูลเชิงโครงสร้างและ progress tracking
- CoreData เป็นทางเลือกหากต้องการควบคุม schema และ migration อย่างละเอียด
- Bundle JSON หรือ SQLite สำหรับ dictionary และ content offline ขนาดใหญ่

### Content Model ที่ควรออกแบบ

- `PaliTerm`: คำบาลี, คำแปล, หมวด, รากศัพท์, metadata
- `GrammarTopic`: หัวข้อไวยากรณ์, เนื้อหา, ตัวอย่าง
- `Flashcard`: หน้าแรก, หลังการ์ด, หมวด, รอบทบทวน
- `TextPassage`: ข้อความบาลี, แหล่งที่มา, ระดับชั้น
- `TokenAnnotation`: คำใน passage, ความหมาย, หน้าที่ทางไวยากรณ์, ลิงก์ไปยังศัพท์
- `ExamPaper`: ปี, ชั้นประโยค, วิชา, ข้อสอบ, เฉลย

### UX และ Typography

- ต้องอ่านตัวอักษรบาลีที่มีพินทุและนิคหิตได้ชัดเจน
- รองรับการปรับขนาดตัวอักษร
- เน้น UI สบายตาสำหรับการอ่านยาว
- ทำ interaction ให้เรียบง่าย เช่น แตะศัพท์ ดูคำแปล บันทึกคำศัพท์เข้าชุดทบทวน

## MVP Scope

MVP ควรเน้นฟีเจอร์ที่ทำให้ผู้เรียนใช้งานได้จริงตั้งแต่วันแรก

- พจนานุกรมบาลี-ไทย Offline
- บทเรียนบาลีไวยากรณ์พื้นฐาน
- Flashcards สำหรับศัพท์และอัพยยศัพท์
- ตารางแจกวิภัตติพื้นฐาน
- หน้าฝึกอ่านธัมมปทัฏฐกถาแบบแตะคำศัพท์ใน passage ตัวอย่าง

## ความเสี่ยงและข้อควรระวัง

- คุณภาพฐานข้อมูลศัพท์เป็นหัวใจสำคัญ หากข้อมูลไม่แม่น แอปจะเสียความน่าเชื่อถือ
- เนื้อหาตำราและข้อสอบควรตรวจสอบสิทธิ์การใช้งานและแหล่งอ้างอิง
- การวิเคราะห์รากศัพท์อัตโนมัติอาจซับซ้อน ควรเริ่มจาก rule-based และ curated data ก่อน
- การจัดรูปแบบอักษรบาลีต้องทดสอบบนอุปกรณ์จริงหลายขนาด
- ข้อสอบและเฉลยต้องแยก metadata ให้ชัดเจน เช่น ปี วิชา ชั้น และแหล่งที่มา

## สรุป

แอปเรียนภาษาบาลีสำหรับบริบทไทยเป็นตลาดเฉพาะที่ต้องให้ความสำคัญกับความถูกต้องของข้อมูล ตำราเรียน และประสบการณ์อ่านบนมือถือ จุดแข็งของแอปควรอยู่ที่ฐานข้อมูลศัพท์ที่ค้นง่าย ระบบช่วยจำที่ใช้งานจริง และเครื่องมือช่วยวิเคราะห์ประโยคที่ทำให้ผู้เรียนเข้าใจโครงสร้างภาษาบาลีได้เร็วขึ้น

การพัฒนาด้วย SwiftUI และ SwiftData เหมาะกับการสร้างประสบการณ์ iOS ที่ลื่นไหล ใช้งาน Offline ได้ และรองรับการต่อยอดไปยังฟีเจอร์ฝึกสอบหรือระบบเรียนแบบรายบุคคลในอนาคต
